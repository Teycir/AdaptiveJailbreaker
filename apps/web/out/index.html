How It Works

01

Configure

Open the admin panel (⚙ top right) to store your OpenRouter API key and worker URL. Keys live only in localStorage.

02

Set Goal

Define the attack goal — the harmful behavior or policy boundary you want to probe for in natural language.

03

The engine runs multi-turn adversarial dialogue, using rollbacks and strategy mutations to maximize success rate.

04

Analyze

Review the full conversation trace, turn-by-turn scores, and rollback tree to understand how and where safety breaks down.

FAQ

AJAR (Adaptive Jailbreak Auditor & Red-teamer) is an automated AI red-teaming tool that uses multi-turn adversarial strategies to probe language model safety boundaries. It is intended exclusively for authorized security research.
Your OpenRouter API key is stored only in your browser's localStorage. It is transmitted per-request to the evaluation worker and is never logged, persisted server-side, or shared with third parties.
Any model available on OpenRouter can be targeted — paste the full OpenRouter model string (e.g. anthropic/claude-3-haiku). The attacker/auditor model is chosen from a curated list of free models to minimize cost.
Crescendo escalates topic sensitivity progressively across turns. ActorAttack uses fictional persona indirection. X-Teaming uses a planner agent to select and adapt attack strategies dynamically.
AJAR is intended for authorized security research only. You must have explicit permission to probe any model or system. Misuse may violate terms of service or applicable law. The authors bear no liability for misuse.
